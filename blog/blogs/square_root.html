SQUARE ROOT

Whether it is in school, or in programming, taking the square root of a number is not a rarity. But, how does this simple, yet widely used algorithm work? What is behind this "√" character?

Introduction


What is taking the square root of a number? Basically, the calculation;
y = √x

can be written as;
y = x * x

In which, you try to find what value for x can be added, to get the value y. Now, there are a lof of reasons to do this, but the biggest usage of the square root is during complicated mathmatical compilations in programming. Whether it is 2D or 3D rendering, most of the time, the square root comes into play, where it is used a lot.

Other than that, it also plays a crucial role in our life.
You can find the usage of square roots in finances, statistics, geometry, physics, engineering or data analysis. Basically, everything that revolves around numbers, is bound to need a square root number.


So, from where did we get this awesome formula though? Well, we gotta go back a lot. In the old times, the Babylonian, Egyptian, and Greek Mathmaticians. Even they used square roots in their time.
The tactics used for determining the square root of a number vary a lot though, as there is always more than just one way of finding out how something works.

However, what we will be focusing on, is the Babylonian method, because it is the most elegant method from all of them (This is not an opinion, it is a fact, deal with it.).




Formula


The Babylonian's had a good idea in mind, when trying to decide what would be a good algorithm, to determine the square root.
As mentioned prior, basically, taking the square root of a number looks like this;
y = x*x

And, you can also write that in a different way;
y / x = x

so, basically, the square root of a number, is the number, divided by the square root itself (duh.). So, a logical formula, would be this;
xn+1 = (xn + (a/xn)) / 2

If you guys didnt get it yet, this is in fact, a recursive function. Lets take it step by step.
The value for "xn", is an initial guess value. In there, you can guess any number you want, doesnt matter. I personally take a (the number we want the root of), and divide it by two, for my first guess, but it doesnt really matter.






Usage

Lets say, our a is 19. We want the square root of that. This is how the function would look;

Step 1.
xn = 19 / 2 = 9.5
xn+1 = (9.5 + (19 / 9.5)) / 2
xn+1 = 5.75

Step 2.
xn+1 = (5.75 + (19 / 5.75)) / 2
xn+1 = 4.527173913

Step 3.
xn+1 = (4.53 + (19 / 4.53)) / 2
xn+1 = 4.3621302428


That is a weird number, but what is the square root of 19 anyway?
Well, the square root is 4.3588989435, which is pretty damn close if you ask me. This method, is not a finite method. You can repeat the recursion, to any extent. Until you get the identical value. Meaning, until xn equals xn+1.

Its as simple as that. This is a quick and easy way to approximate the square root of a number, by simply "running" this recursion three times. But, if we were gonna programm this, how would it look like?






Programm


# 0.000001 perfect value
#
# The bigger the amount of 0 there is
# the more is computational power required,
# because the recursio continues, until
# the statement is met
CLOSE_RANGE = 0.000001

# It compares xn+1 and xn, to determine
# how close the values are together.
# If they are close enough, it
# returns true
def checker(val, num):
# to determine the closeness of the range
# we gotta ignore the negative decimal values.
# So, we just get both positiv, and negative, and
# then compare both
l = val - num
l2 = num - val

return abs(l) < CLOSE_RANGE or abs(l2) < CLOSE_RANGE


def square_root(num):
# s = num / 2, is basically
# as mentioned prior, a starting point.
# s is in this case xn
s = num / 2

# l is our xn+1, and because we havent
# started calculating yet, it is set to 0
l = 0

# recursive function
while True:
# checks whether or not
# xn multiplied by itself,
# equals the number
if checker(s*s, num):
print(s) # print the result
break
else:
# else, it sets the xn to a new value
# and then checks again
l = num / s
if checker(l*l, num):
print(l)
break
else:
# if that didnt work
# it uses the formula from the Babylonians
# and sets xn to that value, to re-start the
# function
s = (s+l)/2

square_root(64) # 8.000000000000101
